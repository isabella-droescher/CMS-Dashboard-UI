CMS Dashboard UI
Design eines CMS- und App-Systems für Events mit Fokus auf
konsistente Nutzererlebnisse

Problem: 
Event-Organisatoren benötigen ein System, um Events effizient zu verwalten, während Nutzer in der App eine konsistente und klare Darstellung erwarten.

Ziel: 
Entwicklung eines konsistenten CMS- und App-Systems mit klarer Informationsstruktur und einheitlicher UX.

Rolle: 
UI/UX Design · Prototyping · Informationsarchitektur

Prozess:
* Analyse der Informationsstruktur
* Erstellung von User Flows für CMS und App
* Wireframing verschiedener Layout-Ansätze
* Entwicklung eines konsistenten UI-Systems in Figma

Lösung:
* CMS: strukturierte Eventverwaltung
* App: klare Event-Übersicht für Nutzer
* einheitliche visuelle Sprache zwischen beiden Systemen

Learnings: 
Fokus auf Konsistenz zwischen zwei Perspektiven (Admin vs. User) verbessert Verständlichkeit und UX deutlich.
