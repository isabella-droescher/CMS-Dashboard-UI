# CMS-App-UI-UX-Design

Combined UI/UX Design Challenge: Events App & CMS

Your task is to design two interconnected screens: 
- an overview screen in the CMS and
- the corresponding screen in the app.
  
# Challenge Overview
Your designs should focus on two perspectives:
1. The CMS: Used by event organizers to create, manage, and update events.
2. The App: Used by customers to explore upcoming events and view event details.

The goal is to create intuitive, recognizable, and practical designs that work seamlessly
together, ensuring both organizers and attendees can easily achieve their objectives.

# Deliverables
1. CMS Overview Screen
- A dashboard interface for managing events.
- Users should be able to view, search, and filter events (e.g., by date, type, or status).
- Include options for adding, editing, and deleting events.
- Display important event details (e.g., name, date, location, and status).
  
2. App Events Overview Screen
- A customer-facing screen displaying a list of events.
- Showcase events with key information such as name, date, and location.
- Enable users to tap on an event to view its detailed screen.
- Highlight upcoming events and use engaging visuals to draw user attention.
  
3. Optional: Add interactivity or show transitions between screens for a more dynamic
presentation.
